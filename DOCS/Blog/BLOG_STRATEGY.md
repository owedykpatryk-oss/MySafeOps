## Anchor text guidance

When linking between articles, vary the anchor text. Don't always use the keyword. Mix:

- Keyword match: "permit to work app UK" (use 30% of links)
- Partial match: "UK permit to work software" or "digital PTW"
- Descriptive: "how confined space permits work"
- Contextual: "the fire watch problem" (links to hot work article)

Google penalises over-optimised anchor text. Natural variation wins.

## Schema.org markup for FAQ snippets

Each article has a FAQ section — mark it up with FAQPage schema so Google shows your answers directly in search results. Example for Article 1:

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [{
    "@type": "Question",
    "name": "Are digital permits to work legally accepted in the UK?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "Yes. HSE guidance (HSG250) recognises electronic permit to work systems where they provide equivalent control and evidence to paper permits."
    }
  }]
}
```

Apply the same pattern for every FAQ in every article. This is one of the biggest SEO wins for B2B content in 2026.

## Expected SEO timeline

- **Month 1:** Articles indexed, initial impressions in Google Search Console
- **Month 2-3:** Long-tail queries start ranking (page 2-3)
- **Month 4-6:** Primary keywords reach page 1 for less competitive terms (confined space permit UK, COSHH register software UK)
- **Month 6-12:** Pillar page (permit to work app UK) competes for top rankings
- **Month 12+:** Cluster authority builds — ranking for queries you didn't explicitly target

SEO is slow. The 7-article cluster is a foundation, not a campaign. Add 1-2 articles per month on adjacent topics (RIDDOR reporting, CDM compliance for small contractors, scaffold inspection app, etc.) to keep the cluster growing.
